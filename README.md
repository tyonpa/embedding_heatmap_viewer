# Embedding Heatmap Viewer

Streamlit で動く、文章埋め込み可視化アプリです。  
CPU 上で埋め込みモデルを切り替えながら、通常文章と足し引き文章ブロックのベクトルを比較できます。

[デモはこちら！](https://emb-heatmap.streamlit.app/)

## Overview

- 埋め込みモデルを `EmbeddingGemma 300M` と `SBERT Multilingual MiniLM` から切り替え可能
- 通常文章をそのままベクトル化して比較
- 複数の短文を `+` / `-` で合成し、1つのベクトルとして比較
- 足し引きブロックを複数作成可能
- ブロック順を並び替え可能
- 足し引きブロックをブロック単位で削除可能
- 任意の2ベクトルの差分ヒートマップを複数ペアで表示可能
- 比較対象ベクトル同士のコサイン類似度を可視化

## Example Use Case

たとえば次のような比較ができます。

- 通常文章: `私は犬が好きです。`
- 足し引きブロック: `+「私」+「犬」+「好き」`

このときアプリでは、次の2つを主に比較できます。

- 元の文 `私は犬が好きです。`
- 分解して合成した `+「私」+「犬」+「好き」`

## UI Structure

### 1. 通常文章

- 文章そのものを比較対象として追加
- 行単位で並び替え可能

### 2. 足し引きブロック

- `+` / `-` を持つ複数文をまとめて1つのベクトルに変換
- ブロックごとに独立して管理
- ブロック単位で並び替え可能
- ブロック単位で削除可能
- `項数で割る` は標準でオン
- `項数で割る` をオンにすると、足し引き後のベクトルを `abs(足し算数 - 引き算数)` で割る

### 3. 差分ヒートマップ

- 比較対象ベクトルから任意の2つを選択
- `左 - 右` の差分ベクトルを表示
- 複数ペアを追加可能
- 色スケールは差分元2ベクトルの `max_abs` を基準に決定

### 4. ヒートマップ

- カラーマップは `bwr`
- 左側に文章ラベルを表示
- 0 を中心とした対称スケールで描画
- 比較表示はブロック順を維持して描画

## How Composition Works

足し引きブロックのベクトルは、各文の埋め込みベクトルを求めたあとで符号付き和として合成します。

```text
+「私」+「犬」-「猫」
```

`項数で割る` をオンにした場合のみ、合成後のベクトルを `abs(足し算数 - 引き算数)` で割ります。

例:

- `+` が 3 件、`-` が 1 件なら `3 - 1 = 2` で割る
- `+` が 2 件、`-` が 5 件なら `5 - 2 = 3` で割る

差が 0 の場合はゼロ除算を避けるため、そのままのベクトルを使います。

## Model Selection

上部のセレクタから次の埋め込みモデルを選べます。

- `google/embeddinggemma-300m`
- `sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2`

どちらも CPU で読み込みます。初回は選んだモデルのダウンロードが発生します。

## Difference Heatmap

比較表示に含まれるベクトルから任意の 2 つを選び、`左 - 右` の差分ベクトルを作成します。

- 複数の差分ペアを設定可能
- 差分表示も `bwr` の 0 基準ヒートマップ
- `vmin` / `vmax` は差分元 2 ベクトルの絶対値最大に合わせて決定

## Cosine Similarity

比較表示に含まれるベクトルについて、コサイン類似度を計算して表示します。

- 上三角ヒートマップ
- 類似度が高い上位ペア

どの通常文章とどの合成ベクトルが近いかを確認しやすくしています。
